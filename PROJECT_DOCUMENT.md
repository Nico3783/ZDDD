In an increasingly interconnected digital environment, modern organizations rely heavily on computer networks to support essential services such as banking, healthcare, telecommunications, industrial automation, and cloud computing. While this connectivity improves efficiency and accessibility, it also increases exposure to cyber threats, especially sophisticated attacks that exploit weaknesses in networked systems. One of the most dangerous categories of cyber threats is the zero-day exploit, which refers to an attack that takes advantage of a previously unknown vulnerability before a patch or signature is available. Because zero-day exploits occur without prior indicators, they are extremely difficult to detect using conventional security tools and often cause significant damage before defenders become aware of the intrusion. 
Traditional intrusion detection systems (IDS) have largely depended on signature-based detection, where network traffic is compared against known malicious patterns. Although effective for known attacks, signature-based systems are limited when facing novel threats and zero-day DoS exploits because they cannot detect malicious activity that does not match existing rules or signatures. This weakness has been widely discussed in intrusion detection literature, where researchers have emphasized that anomaly-based and behaviour-based techniques are necessary for detecting unknown threats (Garcia-Teodoro et al., 2009). 
However, while anomaly-based intrusion detection offers strong potential for identifying zero-day exploits, it introduces its own challenges, such as high false positive rates and difficulty maintaining accuracy under changing network conditions. To overcome these limitations, many researchers have proposed machine learning-based intrusion detection techniques that learn normal network behaviour and identify deviations as suspicious activity. Sommer and Paxson (2010) highlighted that applying machine learning to intrusion detection requires careful consideration due to issues such as real-world traffic variability and the difficulty of collecting representative training data. Nevertheless, the growing availability of benchmark datasets and computational resources has led to increased adoption of machine learning approaches for network intrusion detection. 
Buczak and Guven (2016) provided a broad survey of machine learning and data mining techniques applied in cybersecurity, showing that hybrid detection models often outperform single-method IDS approaches by combining the strengths of anomaly detection and supervised classification. In particular, anomaly detection algorithms such as One-Class SVM, Isolation Forest, and deep learning-based autoencoders have been shown to perform effectively in identifying unknown attack behaviour (Chalapathy & Chawla, 2019). A notable example is the work by Mirsky et al. (2018), who developed Kitsune, an online network intrusion detection system based on an ensemble of autoencoders capable of detecting anomalies in real time. Their work demonstrated that online learning and lightweight deep models can support near real-time intrusion detection, even on resource-constrained environments. These studies collectively demonstrate that combining behavioural monitoring with anomaly detection is a promising direction for detecting zero-day exploits in modern networks. 
Despite existing advances, a key research gap remains in developing detection engines that are both real-time and autonomous, while maintaining high detection accuracy and low false positives under realistic traffic conditions. Many detection approaches in literature are evaluated in offline settings, making them difficult to deploy in real operational networks where traffic is continuous and high-volume. Furthermore, network behaviour evolves over time due to new applications, user activity changes, and infrastructure upgrades, leading to concept drift, which can reduce model effectiveness if systems are not updated regularly. 
Therefore, intrusion detection solutions must not only detect unknown threats but also adapt autonomously to evolving traffic patterns. This project addresses these challenges by developing a real-time autonomous detection engine using a network-based detection approach that integrates anomaly detection for identifying unknown behaviours and supervised classification for recognizing known DoS attack patterns. The system is designed to process high-throughput network traffic in real time using benchmark intrusion datasets such as CICIDS2017, while simulating live traffic conditions to ensure realistic performance evaluation. By adopting efficient machine learning models such as Isolation Forest for anomaly detection and Random Forest for classification, the proposed detection engine aims to provide timely alerts, low detection latency, and improved reliability compared to traditional signature-only IDS methods. Ultimately, this study contributes to improving network security by providing an implementable framework for detecting zero-day DoS exploit behaviour through intelligent, autonomous, and real-time network traffic analysis.

1.5 METHODOLOGY
The methodology adopted for this project follows an experimental and implementation-based approach aimed at developing a real-time autonomous network-based detection engine capable of identifying zero-day exploits. The project is carried out in three major phases: system design, system implementation, and evaluation. 
Since zero-day exploits are previously unseen attacks with no known signatures, the proposed detection engine focuses on behavioural deviations in network traffic using a hybrid model that integrates anomaly detection and classification techniques. The system is designed as a modular pipeline that supports real-time processing, low detection latency, and automated alert generation. This approach ensures the developed prototype is practical, reproducible, and suitable for real-world intrusion detection scenarios. 
Data collection for the project is based on publicly available benchmark network intrusion datasets. The primary dataset used is the CICIDS2017 dataset, which was selected because it provides realistic network traffic containing both benign activities and multiple attack categories, particularly rich DoS profiles, and it is widely accepted in intrusion detection research. The dataset was obtained from the Canadian Institute for Cybersecurity (CIC), where it is made available for academic use. The dataset provides network traffic in extracted flow-based feature format (CSV files), which significantly reduces implementation complexity by eliminating the need for raw packet parsing. Optional datasets such as UNSW-NB15 or CICIDS2018 may also be used for extended evaluation and generalization testing. These datasets were used as the primary source of network traffic records for training, testing, and real-time simulation of the detection engine. 
After dataset acquisition, the data processing stage begins with preprocessing to ensure the traffic records are clean, consistent, and suitable for machine learning. Preprocessing tasks include handling missing values, removing duplicated or corrupted entries, converting labels into standardized formats, and separating benign traffic from attack traffic. Benign records are extracted and used to train the anomaly detection component so that it learns the normal baseline behaviour of network traffic. Numerical features are normalized using scaling techniques such as StandardScaler or MinMaxScaler to ensure that features with large ranges do not dominate model learning. Where categorical features exist, encoding methods are applied to convert them into numerical form. 
Feature selection is then performed to retain the most relevant flow-level characteristics that represent behavioural patterns, such as flow duration, packet counts, byte rates, packet length statistics, inter-arrival time, and TCP flag counts. These selected features are critically important because they reflect traffic behaviours commonly associated with DoS scanning, volume spikes, and abnormal communication patterns, which are key indicators of zero-day exploit activity. 
To meet the real-time requirement of the project without requiring complex live packet capture infrastructure, the system implements real-time traffic simulation using dataset streaming. This is achieved by replaying network traffic dataset records sequentially in a continuous stream, where each record is processed immediately upon arrival. A short time delay or micro-batch mechanism is introduced between records to mimic how traffic arrives in real network environments. This method ensures the system can be tested under realistic real-time constraints while remaining simple to implement and reproducible. The simulation also enables the measurement of detection latency and throughput, which are important indicators of real-time performance. 
The detection engine is implemented as a hybrid model consisting of two components: anomaly detection for identifying unknown or zero-day behaviours, and classification for labeling known attacks. The anomaly detection component is implemented using Isolation Forest, a machine learning algorithm designed to identify rare patterns by isolating anomalies using random partitioning. Isolation Forest is chosen due to its efficiency, suitability for high-dimensional traffic features, and low computational overhead, making it ideal for real-time processing. The model is trained primarily on benign traffic to learn normal behaviour, and during detection it assigns an anomaly score to each incoming record. Records with anomaly scores exceeding a defined threshold are considered suspicious and forwarded to the classification component. 
The classification component is implemented using Random Forest, a supervised learning algorithm that combines multiple decision trees to improve prediction accuracy and reduce overfitting. Random Forest is chosen because it performs well on intrusion detection datasets, handles nonlinear feature relationships effectively, and provides interpretable feature importance scores. The classifier is trained using labeled traffic records containing both benign and known DoS attack classes, allowing it to predict the most likely attack category for suspicious traffic. 
The tools used for system development and experimentation are selected to ensure simplicity and efficiency. Python is used as the main programming language due to its strong support for machine learning and data processing. Key libraries include pandas and numpy for handling and processing the dataset, scikit-learn for implementing the Isolation Forest and Random Forest models, joblib for saving and loading trained models, and matplotlib for producing evaluation graphs and visualizations. The real-time simulation pipeline is implemented directly in Python using sequential file streaming and timing functions, ensuring the engine behaves like a real-time detection system without requiring advanced streaming platforms such as Apache Kafka. For optional analysis and verification, tools such as Wireshark may be used to inspect raw traffic where PCAP files are available. A baseline comparison can be carried out using a signature-based intrusion detection system such as Suricata or Snort to demonstrate the advantage of anomaly-based detection in identifying zero-day behaviours. 
During system execution, the real-time detection workflow operates as follows: dataset records are streamed into the engine, pre-processed, and converted into scaled feature vectors. The Isolation Forest model evaluates each record and generates an anomaly score. If the anomaly score exceeds the predefined threshold, the record is classified by the Random Forest model to determine whether it resembles known attack patterns. When suspicious traffic is detected, the system automatically generates an alert containing details such as timestamp, anomaly score, predicted label, and severity level. Alerts are logged in structured formats such as CSV or JSON to support later analysis and reporting. This automated alerting ensures that the system functions autonomously with minimal human intervention. 
The evaluation phase measures both detection effectiveness and real-time performance. The system is tested using the benchmark datasets in a simulated real-time environment. Detection effectiveness is measured using standard intrusion detection metrics including accuracy, precision, recall (detection rate), F1-score, and false positive rate. Real-time performance is evaluated using detection latency, which measures the time taken from record arrival to alert generation, and throughput, which measures the number of records processed per second. The results are compared against baseline approaches, particularly signature-only detection methods, to demonstrate the system’s ability to detect unknown threats. Overall, this methodology ensures that the proposed detection engine meets the project objectives by combining robust data collection from standard datasets, structured preprocessing and feature engineering, hybrid machine learning detection, real-time simulation, and comprehensive evaluation using standard performance metrics.

2.2 Overview of Intrusion Detection Systems 
Intrusion Detection Systems (IDS) are security mechanisms designed to monitor network or system activities and identify malicious behavior. IDS are generally classified into two main types: signature-based (misuse-based) and anomaly-based systems. According to the National Institute of Standards and Technology (NIST), IDS tools operate by collecting event information and analyzing it to identify suspicious patterns or policy violations (Scarfone & Mell, 2007). IDS remain essential in cybersecurity because they serve as a detection layer that complements preventive measures such as firewalls and access controls. 
Signature-based IDS detect intrusions by matching observed traffic against a database of known attack patterns. This approach is widely used in industry because it provides high accuracy for known threats and low false alarm rates. However, signature-based systems cannot detect novel threats such as zero-day exploits because they depend entirely on previously known signatures. Garcia-Teodoro et al. (2009) emphasized that signature-based IDS are limited in their ability to identify new attacks and are therefore inadequate for modern evolving threat landscapes. The inability to detect unknown attacks has led to a shift towards anomaly-based and intelligent IDS techniques. 

2.3 Zero-Day Exploits and Their Detection Challenges 
Zero-day exploits represent one of the most serious cybersecurity threats because they exploit vulnerabilities before defenders can develop patches or signatures. Bilge and Dumitras (2012) conducted an empirical study on real-world zero-day attacks and demonstrated that attackers often exploit vulnerabilities for long periods before public disclosure. This finding highlights the need for detection systems that do not depend on prior knowledge of vulnerabilities. 
The detection of zero-day exploits is challenging because their behaviours may resemble normal traffic, and attackers often attempt to mimic legitimate patterns to evade detection. Additionally, zero-day exploits may appear as low-volume attacks, targeted attacks, or stealthy intrusions that do not generate obvious network anomalies. When transitioning to DoS contexts, novel amplification or protocol-draining logic can bypass traditional thresholds. These factors make it difficult for traditional IDS to reliably detect zero-day threats. 
2.4 Anomaly-Based Network Intrusion Detection 
Anomaly-based intrusion detection is based on identifying deviations from normal network behaviour. Garcia-Teodoro et al. (2009) described anomaly-based detection as a method that builds a model of normal activity and flags patterns that differ significantly from this model. Patcha and Park (2007) also provided a comprehensive overview of anomaly detection techniques and identified statistical methods, machine learning, and knowledge-based approaches as major categories. 
Anomaly-based IDS are particularly suitable for zero-day exploit detection because they do not require predefined attack signatures. However, one of the major limitations is that anomaly detection systems often generate high false positive rates. Legitimate but rare network behaviours can be incorrectly classified as attacks. This limitation is especially problematic in real network environments where traffic patterns are dynamic and unpredictable. 


2.5 Machine Learning Techniques for Network Intrusion Detection 
Machine learning has become widely applied in intrusion detection due to its ability to learn patterns from large datasets and generalize to unseen behaviours. Buczak and Guven (2016) surveyed machine learning methods used for intrusion detection and concluded that supervised learning approaches such as decision trees, SVM, k-NN, and ensemble models provide strong performance for known attacks. They also noted that unsupervised and semi-supervised approaches are more suitable for detecting unknown threats. 
Liao and Vemuri (2002) demonstrated the use of the k-Nearest Neighbour (k-NN) classifier for intrusion detection. While k-NN can achieve good classification accuracy, it has limitations in real-time detection due to high computational cost when processing large datasets, as it requires comparison against many training samples. Isolation Forest, introduced by Liu et al. (2008), is an unsupervised anomaly detection technique designed to isolate anomalies by randomly partitioning data. Isolation Forest is computationally efficient and performs well for high-dimensional datasets. However, its limitation is that threshold selection can be difficult, and poor threshold tuning can lead to increased false positives. 
2.6 Deep Learning Approaches for Intrusion Detection 
Deep learning has gained attention for intrusion detection due to its ability to automatically learn complex representations from data. Javaid et al. (2016) proposed a deep learning-based IDS that demonstrated improved performance over traditional machine learning methods. Chalapathy and Chawla (2019) surveyed deep learning methods for anomaly detection and showed that autoencoders, recurrent neural networks, and convolutional models can detect anomalies effectively by learning feature representations. 
A major contribution in online anomaly detection is Kitsune, proposed by Mirsky et al. (2018) . Kitsune uses an ensemble of autoencoders for online intrusion detection and demonstrated real-time detection capability. Despite its strong performance, Kitsune and similar deep learning methods face limitations such as the need for careful tuning, risk of overfitting, and higher computational requirements compared to traditional lightweight algorithms like Isolation Forest. Additionally, deep learning models often require large labeled datasets for training, which may not always be available. They may also be difficult to interpret, reducing transparency and trust for security analysts. 
2.7 Hybrid Intrusion Detection Systems 
Hybrid intrusion detection combines multiple detection strategies, typically anomaly detection and misuse detection, to improve overall performance. Kim et al. (2014) proposed a hybrid intrusion detection method integrating anomaly detection with misuse detection and found that hybrid systems can reduce false positives and improve detection accuracy compared to single-method approaches. Buczak and Guven (2016) also identified hybrid systems as a promising direction, as they can detect both known and unknown attacks. However, hybrid systems are often more complex to implement and may require careful coordination between models to avoid inconsistent outputs. Some hybrid IDS also suffer from increased computational cost, making real-time deployment challenging. 
2.8 Real-Time Intrusion Detection and Streaming Challenges 
Real-time intrusion detection is essential for preventing the spread of attacks. However, Sommer and Paxson (2010) argued that many machine learning approaches fail in real-world intrusion detection because they assume closed-world environments where training and testing data distributions remain stable. In reality, network traffic is noisy, highly variable, and evolves continuously. 
Ring et al. (2019) surveyed network intrusion detection datasets and identified a major limitation: many datasets do not fully represent modern real-world traffic. Some datasets are outdated, imbalanced, or fail to capture encrypted traffic patterns. This limitation affects the generalization ability of intrusion detection models trained on such datasets. Ahmed et al. (2016) highlighted that real-time intrusion detection faces challenges including scalability, feature extraction cost, and high false positive rates. They emphasized the need for lightweight models that can operate in streaming environments with minimal delay. 

2.9 Concept Drift and Adaptive Learning 
Concept drift occurs when network traffic behaviour changes over time. If models are not updated, detection performance can degrade. Although many IDS models are trained offline, real-world systems require adaptation to maintain accuracy. The ability to update models autonomously is increasingly recognized as critical, especially for anomaly-based systems. However, adaptive learning introduces new risks. If model updates are performed without verification, attackers can poison training data by injecting malicious patterns into the normal traffic stream. Therefore, model adaptation must be carefully controlled to avoid learning malicious behaviours as normal. 
2.10 Research Gap
From the reviewed literature, several gaps are identified. Signature-based IDS remain ineffective against zero-day exploits due to reliance on known signatures (Scarfone & Mell, 2007). Anomaly-based IDS can detect unknown attacks but often suffer from high false positives (Garcia-Teodoro et al., 2009; Patcha & Park, 2007). Machine learning and deep learning approaches improve detection capabilities but face challenges related to real-time scalability, computational overhead, and concept drift (Sommer & Paxson, 2010; Mirsky et al., 2018). Hybrid IDS approaches reduce weaknesses of single methods but increase system complexity and real-time deployment challenges (Kim et al., 2014; Buczak & Guven, 2016). 
Additionally, many studies evaluate their systems offline and fail to implement realistic real-time detection pipelines. Therefore, there is a need for a detection engine that is lightweight, real-time, autonomous, and capable of detecting both known and unknown DoS threats using network-based traffic features. This project addresses this gap by implementing a hybrid detection engine using efficient machine learning algorithms (Isolation Forest and Random Forest) and evaluating it under real-time simulation using benchmark intrusion datasets.

INTRODUCTION
1.1 Introduction
Denial of Service (DoS) attacks represent one of the most persistent and disruptive categories of cyber threats targeting modern networked infrastructure. These attacks aim to exhaust the computational resources, bandwidth, or availability of a target system, rendering it unable to serve legitimate users. While traditional DoS attacks based on known techniques such as SYN flood, UDP flood, and ICMP flood can be mitigated using established countermeasures, a far more dangerous class of threat has emerged: zero-day DoS exploits. These attacks leverage previously unknown vulnerabilities in network protocols, operating systems, or application layers to launch DoS campaigns for which no signature, rule, or prior mitigation strategy exists. The stealthy and novel nature of zero-day DoS exploits means they can evade conventional intrusion detection systems (IDS) entirely, resulting in prolonged outages, service degradation, financial losses, and reputational damage to organizations.
Traditional IDS approaches rely heavily on signature-based detection, comparing observed traffic against a database of known attack patterns. While effective for catalogued DoS attack signatures, these systems are fundamentally incapable of detecting zero-day DoS variants that have no known signature. Anomaly-based detection approaches offer a promising alternative by modelling the statistical properties of normal traffic and flagging significant deviations. However, they are often challenged by high false positive rates, especially in dynamic DoS traffic environments where traffic volume and patterns fluctuate rapidly. This creates a critical gap: the need for a detection engine that can autonomously and accurately identify zero-day DoS exploit behaviour in real time without depending on predefined signatures.
This project proposes the development of a Real-Time Autonomous Anomaly Detection Engine specifically designed to identify zero-day exploits manifested within DoS attack traffic. The system adopts a hybrid machine learning approach, integrating Isolation Forest for unsupervised anomaly detection with Random Forest for supervised classification of known DoS attack types. By training on the behavioural characteristics of normal network traffic and benign baselines, the engine learns to distinguish subtle anomalous patterns indicative of novel DoS exploit activity. A real-time simulation pipeline based on the CICIDS2017 benchmark dataset, which contains realistic DoS and DDoS attack scenarios alongside benign traffic, is implemented to evaluate detection performance under realistic streaming conditions. The goal is to achieve timely, accurate, and autonomous detection of zero-day DoS exploits while maintaining low false positive rates and minimal detection latency.
Sommer and Paxson (2010) highlighted that deploying machine learning in real-world network intrusion detection requires careful attention to operational constraints such as traffic variability and concept drift. Within the DoS domain, these challenges are amplified because attack traffic can vary dramatically in volume, packet size distribution, and protocol composition depending on the specific exploit used. Mirsky et al. (2018) demonstrated through Kitsune that lightweight ensemble anomaly detection models can operate in near real-time environments, making them practical candidates for DoS detection pipelines. Buczak and Guven (2016) established that hybrid detection models combining anomaly detection with classification consistently outperform single-method approaches on intrusion detection benchmarks. These insights directly inform the design of the proposed engine, which is tailored specifically to the characteristics of DoS traffic and zero-day behavioural anomalies within that domain.
The research gap that this project addresses is the absence of a lightweight, real-time, and domain-specific detection engine focused exclusively on zero-day exploit identification within DoS attack contexts. Most existing IDS research targets general network intrusion detection, without specializing in the traffic dynamics particular to DoS-based exploits. This project contributes a focused, implementable framework that narrows this gap by combining domain-targeted feature engineering, efficient machine learning models, and a real-time simulation infrastructure to produce a detection engine suitable for deployment in operational network security environments.
1.2 Background of the Study
Denial of Service attacks have been a recurring threat to internet-connected systems since the early days of networking. A DoS attack disrupts the availability of a target’s resources by overwhelming it with malicious traffic or by exploiting protocol weaknesses to crash or impair the target system. Distributed Denial of Service (DDoS) attacks extend this model by coordinating traffic from thousands of compromised machines, dramatically amplifying the attack’s impact. High-profile targets including financial institutions, government agencies, healthcare systems, and critical infrastructure have suffered significant disruptions due to DoS and DDoS campaigns.
While established DoS attack types such as TCP SYN flood, HTTP flood, UDP amplification, and Slowloris attacks are well-documented and largely detectable by modern IDS, the threat landscape has evolved significantly. Attackers now weaponize zero-day vulnerabilities — previously unknown flaws in network stacks, server software, or communication protocols — to craft DoS exploits for which no defense rule or signature has been created. These zero-day DoS exploits are particularly dangerous because defenders have no prior knowledge of the attack vector, making timely detection extremely difficult using conventional tools.
Signature-based IDS such as Snort and Suricata maintain large rule sets for known DoS patterns and can detect them reliably. However, they are structurally incapable of detecting zero-day DoS exploits because they require a known signature to trigger an alert. This limitation has driven research into anomaly-based and machine learning–based IDS approaches that build models of normal traffic behaviour and flag significant statistical deviations. Within the DoS context, anomalous behaviour typically manifests as unusual traffic rates, abnormal packet size distributions, unexpected protocol usage, sudden increases in connection requests from specific sources, or irregular flag combinations in TCP packets.

Despite the availability of intrusion detection datasets such as CICIDS2017 that include labelled DoS attack traffic, most machine learning–based IDS systems are designed and evaluated for general network intrusion detection rather than being specifically tuned for DoS and zero-day DoS exploit scenarios. The unique traffic characteristics of DoS attacks — including extremely high volume, rapid packet generation, and targeted protocol abuse — require feature engineering and model selection decisions that differ from general intrusion detection contexts. Therefore, this project addresses the need for a detection engine that is purpose-built for real-time zero-day DoS exploit identification using machine learning.
1.3 Motivation for the Study
Several critical realities motivate this research:
    • Zero-day DoS exploits are undetectable by conventional IDS: Since zero-day DoS attacks exploit previously unknown vulnerabilities, signature-based IDS generate no alerts, leaving systems exposed until post-incident signature development.
    • DoS attacks cause immediate and severe service disruption: Unlike data exfiltration attacks that may go unnoticed for weeks, DoS attacks degrade or eliminate service availability instantly, making real-time detection a strict operational requirement rather than a desirable feature.
    • The DoS threat surface is expanding rapidly: The proliferation of IoT devices, cloud-native applications, and high-speed networks has greatly increased the attack surface for DoS campaigns, while also increasing the volume of traffic that detection systems must process in real time.
    • Anomaly detection for DoS generates excessive false positives: Legitimate traffic surges such as flash crowds can resemble DoS traffic patterns, causing anomaly-based detectors to generate excessive false positives unless the detection model is carefully tuned to DoS-specific behavioural features.
    • Autonomous detection is essential for operational scalability: Security operations teams cannot manually review the volume of events generated during an active DoS campaign. The system must autonomously triage and alert on genuine zero-day DoS anomalies with minimal analyst intervention.
    • Existing research lacks DoS-domain specificity: Most published ML-based IDS research addresses general intrusion detection. A detection engine purpose-built for the behavioural characteristics of zero-day DoS exploits fills a meaningful gap in both academic research and practical security tooling.
1.4 Aim and Objectives
Aim
The aim of this project is to develop a real-time autonomous anomaly detection engine capable of identifying zero-day exploits embedded within Denial of Service (DoS) attack traffic, using a hybrid machine learning approach applied to network flow–based features extracted from benchmark intrusion detection datasets under real-time simulation conditions.
Objectives
The specific objectives of this project are to:
    i. Conduct a comprehensive review of DoS attack taxonomies, zero-day exploit characteristics, and existing ML-based IDS approaches applicable to DoS traffic analysis.
    ii. Engineer and select DoS-domain-specific network flow features from the CICIDS2017 dataset that effectively capture the behavioural indicators of zero-day DoS exploit activity, including traffic rate metrics, packet size statistics, TCP flag distributions, and inter-arrival time patterns.
    iii. Develop an anomaly detection model using Isolation Forest trained exclusively on benign traffic baselines to detect statistically anomalous DoS-context traffic flows that may indicate zero-day exploit activity.
    iv. Implement a supervised Random Forest classification component to categorize flagged anomalies into known DoS attack types or novel/unknown attack patterns, enabling triage and prioritization of alerts.
    v. Design and implement a real-time DoS traffic simulation pipeline that streams CICIDS2017 flow records continuously, simulating the high-throughput, high-volume conditions characteristic of active DoS attack environments.
    vi. Evaluate the detection engine’s performance using standard intrusion detection metrics — accuracy, precision, recall, F1-score, false positive rate, and detection latency — and compare results against signature-only baseline approaches to quantify the advantage of the proposed autonomous anomaly detection method in the DoS exploit detection context.
1.5 Methodology
The methodology adopted for this project follows an experimental and implementation-based approach structured around five sequential phases: dataset acquisition and domain scoping, DoS-specific feature engineering and preprocessing, hybrid model development, real-time simulation pipeline construction, and comprehensive evaluation. The research is anchored in the specific behavioural characteristics of DoS attack traffic, including the high-volume, high-rate, and protocol-abusive nature of such attacks, to ensure that the detection engine is purpose-built for the DoS zero-day exploit context rather than general network intrusion.
Phase 1: Dataset Acquisition and Domain Scoping
The primary dataset used is the CICIDS2017 dataset, selected because it contains labelled network flows for realistic DoS and DDoS attack scenarios including Slowloris, Slowhttptest, Hulk, GoldenEye, and Heartbleed, alongside substantial benign traffic. These DoS attack categories represent a representative range of known DoS techniques against which the anomaly detection model can be calibrated and evaluated. The dataset is sourced from the Canadian Institute for Cybersecurity in CSV flow-feature format, eliminating the need for raw packet parsing. For extended generalization evaluation, the UNSW-NB15 dataset, which also contains DoS attack traffic, is considered as a secondary validation dataset. Only traffic flows relevant to the DoS and DDoS attack categories and their benign counterparts are retained from the full CICIDS2017 dataset to maintain domain focus.
Phase 2: DoS-Specific Feature Engineering and Preprocessing
Preprocessing begins with data cleaning to handle missing values, remove corrupted entries, normalize label formats, and separate benign from attack traffic flows. Given the DoS domain focus, feature selection prioritizes traffic characteristics that are most discriminative for DoS exploit behaviour. Selected features include: flow duration, total forward and backward packet counts, forward and backward packet length statistics (mean, standard deviation, maximum, minimum), flow bytes per second, flow packets per second, inter-arrival time statistics for forward and backward flows, TCP flag counts (SYN, FIN, RST, ACK, PSH, URG), active and idle time statistics, and subflow byte and packet counts. These features capture the volume-based, rate-based, and protocol-behavioural indicators that characterize DoS traffic. Features unrelated to DoS traffic dynamics are excluded to prevent noise from diluting the anomaly detection model’s sensitivity. All numerical features are normalized using StandardScaler. Label encoding converts multi-class attack labels into binary format (benign versus attack) for the anomaly detection phase and into multi-class format for the classification phase.
Phase 3: Hybrid Machine Learning Model Development
The detection engine implements a two-stage hybrid model. In the first stage, an Isolation Forest model is trained exclusively on benign DoS-domain traffic flows. Isolation Forest isolates anomalies by recursively partitioning the feature space using random splits, producing an anomaly score for each incoming flow record. Records whose anomaly scores exceed a calibrated threshold are identified as potentially anomalous and forwarded to the second stage. The contamination parameter of the Isolation Forest is tuned based on the proportion of expected anomalous traffic in the DoS domain, and threshold optimization is performed using Receiver Operating Characteristic (ROC) curve analysis on a held-out validation set. In the second stage, a Random Forest classifier trained on labelled DoS attack and benign flow records predicts whether flagged anomalies match known DoS attack categories or represent novel, previously unseen exploit behaviour. The combination of these two stages allows the engine to detect zero-day DoS exploits — which appear as anomalies that do not match any known DoS attack class signature — while also correctly categorizing known DoS attacks. Trained model artefacts are serialized using joblib for deployment in the real-time pipeline.
Phase 4: Real-Time DoS Traffic Simulation Pipeline
Since live capture of zero-day DoS exploit traffic is not feasible in an academic project environment, real-time detection is simulated using dataset streaming. The pipeline reads CICIDS2017 DoS flow records sequentially and injects them into the detection engine at configurable rates using micro-batch dispatch, simulating the high-throughput conditions of an active DoS attack. The pipeline processes each arriving flow record through the preprocessing module, generates a scaled feature vector, evaluates it against the Isolation Forest anomaly scorer, and if flagged, routes it to the Random Forest classifier. Detection latency is measured as the elapsed time from flow record arrival to alert generation. When a record is classified as anomalous, the system generates a structured alert logged in JSON format, containing the timestamp, flow-level features, anomaly score, predicted label, and a severity classification (low, medium, high) based on anomaly score magnitude. This alert log constitutes the primary operational output of the detection engine.
Phase 5: Evaluation Framework
The detection engine is evaluated along two dimensions: detection effectiveness and real-time performance. Detection effectiveness is measured using accuracy, precision, recall (detection rate), F1-score, and false positive rate computed against the ground-truth labels in the CICIDS2017 DoS test subset. Particular emphasis is placed on recall for zero-day-like attack flows (those categorized as novel or unknown by the classifier) and on false positive rate for benign traffic, since false positives in a DoS detection context can disrupt legitimate traffic mitigation decisions. Real-time performance is measured as average detection latency per flow record and total throughput (records per second). Results are compared against a signature-only baseline implemented using Suricata rules for known DoS attack signatures, demonstrating the proposed engine’s ability to detect zero-day DoS exploits that the signature-based baseline cannot identify. Python is used throughout with pandas, numpy, scikit-learn, matplotlib, and seaborn as the primary libraries.
1.6 Expected Outcomes and Impact
This project is expected to produce a functional, real-time autonomous anomaly detection engine that can identify zero-day exploits within Denial of Service attack traffic using machine learning. The engine will demonstrate measurable improvements over signature-only detection approaches in identifying novel DoS attack behaviours not covered by existing rule sets. Operationally, the detection engine will provide a practical framework for early-stage DoS zero-day threat identification, reducing the window of exposure between initial exploit deployment and defender awareness. The project will also produce a documented and reproducible feature engineering methodology tailored to DoS traffic analysis, which can inform future research in DoS-domain-specific IDS development. More broadly, this work contributes to the advancement of autonomous, intelligent security monitoring systems capable of operating effectively against the evolving and unpredictable nature of zero-day Denial of Service threats.

CHAPTER TWO
LITERATURE REVIEW
This chapter reviews existing research relevant to the development of a real-time autonomous anomaly detection engine for identifying zero-day exploits within Denial of Service (DoS) attack traffic using machine learning. The review is structured around six thematic areas: DoS attack taxonomy and zero-day exploit characteristics, intrusion detection system architectures, anomaly-based detection techniques in the DoS domain, machine learning methods applied to DoS intrusion detection, hybrid detection systems, and real-time detection challenges specific to high-volume DoS traffic environments. The chapter concludes by identifying research gaps that justify the proposed system.
2.1 Denial of Service Attacks: Taxonomy and Zero-Day Exploit Characteristics
Denial of Service attacks are broadly classified into volumetric attacks, protocol exploitation attacks, and application-layer attacks. Volumetric attacks overwhelm network bandwidth with large traffic volumes, as seen in UDP flood and ICMP flood techniques. Protocol attacks exploit weaknesses in network protocol implementations to consume server resources, with TCP SYN flood being the canonical example. Application-layer attacks target specific services with legitimate-appearing but resource-intensive requests, as in Slowloris and HTTP flood attacks. Mirkovic and Reiher (2004) provided an influential taxonomy of DDoS attacks and defense mechanisms, establishing that the diversity of DoS attack techniques makes signature-based detection inherently incomplete.
Zero-day DoS exploits represent a specialized and particularly dangerous subclass of DoS attacks. These attacks leverage vulnerabilities in network protocol implementations, operating system networking stacks, or application servers that have not yet been publicly disclosed or patched. Because no signature, patch, or mitigation rule exists at the time of their deployment, zero-day DoS exploits can evade all signature-based defenses and remain active until the underlying vulnerability is discovered and addressed. Bilge and Dumitras (2012) demonstrated empirically that zero-day vulnerabilities are actively exploited for months before public disclosure, emphasizing the critical importance of behavioral detection mechanisms that do not depend on prior vulnerability knowledge. In the DoS context, zero-day exploits may manifest as unusual traffic patterns — unexpected protocol field combinations, atypical inter-arrival time distributions, novel resource consumption profiles — that deviate measurably from established benign traffic baselines.
2.2 Overview of Intrusion Detection Systems in the DoS Context
Intrusion Detection Systems (IDS) are security mechanisms designed to monitor network traffic or system activity and identify suspicious behaviour. Scarfone and Mell (2007) provided the foundational NIST classification of IDS into signature-based (misuse-based) and anomaly-based categories. In the DoS domain, signature-based IDS such as Snort and Suricata maintain extensive rule sets for known DoS attack signatures, including SYN flood detection via connection rate thresholds and application-layer flood detection via HTTP request rate rules. These systems are highly accurate for known DoS attack types but are fundamentally unable to detect zero-day DoS exploits because they rely entirely on pre-existing signature databases.
Garcia-Teodoro et al. (2009) emphasized that anomaly-based IDS address this limitation by modelling normal traffic behaviour and detecting statistically significant deviations. In the DoS context, normal traffic behaviour is characterized by stable traffic rates, typical packet size distributions, balanced TCP flag proportions, and consistent connection patterns. A DoS attack — particularly a zero-day variant — disrupts these statistical properties in characteristic ways, creating anomalies detectable through behavioral analysis. However, Garcia-Teodoro et al. also noted that anomaly-based IDS frequently suffer from elevated false positive rates, especially when traffic patterns fluctuate due to legitimate load spikes, which can closely resemble early-stage DoS activity.
2.3 Anomaly-Based Detection Techniques for DoS Traffic
Anomaly-based detection for DoS traffic has been investigated through multiple methodological approaches. Statistical methods using traffic rate monitoring, entropy analysis of source IP distributions, and packet size histograms have been shown to detect volumetric DoS attacks with reasonable accuracy. Patcha and Park (2007) provided a comprehensive survey of anomaly detection techniques, categorizing approaches into statistical, machine learning, and knowledge-based methods, and concluded that machine learning approaches offer the greatest flexibility and adaptability for detecting unknown attack behaviours.
Isolation Forest, introduced by Liu et al. (2008), is particularly well-suited for DoS-domain anomaly detection due to its efficiency with high-dimensional data and its ability to identify rare patterns without requiring labeled anomaly examples during training. Isolation Forest builds an ensemble of isolation trees that isolate anomalies using fewer random partitions than normal instances, because anomalies are rare and statistically different from the majority of training data. In the DoS detection context, zero-day exploit traffic — characterized by unusual combinations of traffic rate, packet size, and protocol behaviour — produces short isolation path lengths, yielding high anomaly scores. The primary challenge in applying Isolation Forest to DoS traffic is threshold calibration: the contamination parameter must be tuned to reflect the expected proportion of anomalous flows in the traffic stream, a value that varies significantly between DoS attack scenarios.
Ahmed et al. (2016) surveyed network anomaly detection techniques and highlighted that flow-level feature extraction — as opposed to packet-level analysis — enables more scalable anomaly detection in high-volume DoS environments. Flow-level features summarize traffic behaviour over connection intervals, reducing the per-record processing cost significantly while retaining the discriminative information needed to distinguish normal from anomalous DoS-context traffic.
2.4 Machine Learning Techniques for DoS Intrusion Detection
Buczak and Guven (2016) conducted a comprehensive survey of machine learning methods applied to cybersecurity intrusion detection and concluded that ensemble methods, particularly Random Forest, consistently achieve strong classification performance on network intrusion detection benchmarks including DoS attack categories. Random Forest’s robustness to feature correlation, its ability to handle the nonlinear relationships between traffic features and attack labels, and its built-in feature importance scoring make it well suited for the DoS classification component of a hybrid detection engine. The classifier learns to distinguish between known DoS attack types — such as the volumetric versus application-layer distinctions encoded in CICIDS2017 flow labels — and benign traffic, providing a reliable secondary triage layer for flows flagged by the anomaly detector.
Liao and Vemuri (2002) demonstrated the utility of k-Nearest Neighbour (k-NN) classification for intrusion detection, achieving reasonable detection rates for known attack types. However, k-NN is computationally unsuitable for high-volume DoS detection because its prediction time scales linearly with training set size, making it impractical for real-time processing of the high-throughput traffic streams characteristic of active DoS attacks. Random Forest, in contrast, provides prediction in approximately O(log n) time per tree and can be parallelized across trees, making it far more suitable for real-time DoS classification.
Javaid et al. (2016) proposed a deep learning-based intrusion detection system and reported improved detection rates compared to traditional classifiers on the NSL-KDD dataset. Chalapathy and Chawla (2019) surveyed deep learning methods for anomaly detection and demonstrated that autoencoders can effectively learn normal traffic representations and detect deviations in unsupervised settings. While deep learning approaches offer strong representational capacity, they carry significant computational overhead that limits their applicability in real-time DoS detection on resource-constrained hardware. For the proposed engine, the computational efficiency of Isolation Forest and Random Forest is prioritized over the marginal accuracy gains achievable with deep learning, in order to meet the real-time latency requirements of DoS detection.
2.5 Hybrid Intrusion Detection Systems for DoS Detection
Hybrid intrusion detection systems that combine anomaly-based and misuse-based (signature-based) detection have been shown to outperform single-method systems on intrusion detection benchmarks. Kim et al. (2014) proposed a hybrid IDS integrating anomaly detection with misuse detection and demonstrated that the combination reduces false positives while maintaining high detection rates for both known and unknown attacks. In the DoS context, this hybrid design is particularly valuable: signature-based detection handles known DoS attack types with high precision and low false positive rates, while anomaly detection covers novel or zero-day DoS exploits that no signature rule addresses. The proposed engine extends this hybrid paradigm by replacing the signature component with a supervised machine learning classifier (Random Forest), which provides more adaptable coverage of known DoS attack variations than static rule-based signatures.
Mirsky et al. (2018) developed Kitsune, an online network intrusion detection system using an ensemble of autoencoders, and demonstrated that lightweight ensemble anomaly detection models can operate in near real-time environments, even on hardware-constrained platforms. Kitsune’s architecture provides important validation for the feasibility of real-time anomaly detection in high-throughput network traffic contexts, including DoS scenarios. However, Kitsune was designed as a general-purpose IDS rather than a DoS-specific detection engine, and its autoencoder-based approach requires more computational resources than Isolation Forest for equivalent anomaly scoring throughput.
2.6 Real-Time Detection Challenges in High-Volume DoS Traffic Environments
Sommer and Paxson (2010) argued that the real-world deployment of machine learning–based IDS is significantly more challenging than laboratory evaluation suggests, due to traffic distribution mismatch between training and deployment environments, the high cost of false positives in operational networks, and the need for models to generalize to previously unseen traffic patterns. In the DoS context, these challenges are intensified by the extreme traffic volume variations that occur during an active attack. During a volumetric DoS attack, traffic rates can increase by orders of magnitude within seconds, creating a non-stationary traffic distribution that can challenge anomaly detection models trained on normal-volume baselines.
Ring et al. (2019) surveyed network intrusion detection datasets and identified that many benchmarks fail to realistically represent modern DoS traffic, particularly encrypted DoS attack traffic and distributed DoS campaigns coordinated across multiple attack vectors. The CICIDS2017 dataset addresses many of these limitations by including realistic multi-type DoS attack traffic generated from live network infrastructure, making it the most suitable available benchmark for evaluating DoS-domain IDS systems. Nevertheless, limitations in dataset representativeness must be acknowledged when interpreting evaluation results, as the detection engine’s generalization to real-world zero-day DoS exploits — which by definition are not represented in any existing dataset — depends on the degree to which novel attack flows deviate from the learned normal traffic baseline.
Concept drift is a particularly acute challenge for DoS detection models. Network traffic characteristics change over time due to new applications, evolving user behaviour, infrastructure changes, and seasonal traffic patterns. An anomaly detection model trained on a historical benign traffic baseline may generate increasing false positive rates as normal traffic patterns shift away from the training distribution. Ahmed et al. (2016) emphasized that real-time DoS detection systems must incorporate mechanisms for model recalibration to maintain detection effectiveness over operational lifetimes. The proposed engine acknowledges this limitation and identifies periodic model retraining using updated benign traffic samples as a direction for future operational enhancement.
2.7 Research Gap
The reviewed literature reveals a consistent and significant research gap: while substantial research exists on general network intrusion detection using machine learning, no published system has been specifically designed and evaluated as a real-time autonomous anomaly detection engine targeting zero-day exploit identification within the Denial of Service attack domain. Signature-based DoS IDS remain blind to zero-day exploits by design (Scarfone & Mell, 2007). General anomaly-based IDS address unknown threats but are not optimized for the specific traffic dynamics — high volume, high rate, protocol abuse — that characterize DoS attacks (Garcia-Teodoro et al., 2009; Patcha & Park, 2007). Machine learning–based IDS research primarily addresses multi-class general intrusion classification rather than DoS-specific anomaly detection (Buczak & Guven, 2016). Real-time IDS research highlights deployment challenges but does not produce DoS-domain-specific implementations (Sommer & Paxson, 2010; Ahmed et al., 2016). Hybrid IDS improve upon single-method approaches but remain oriented toward general intrusion detection rather than the DoS exploit context (Kim et al., 2014; Mirsky et al., 2018).
This project directly addresses this gap by developing and evaluating a hybrid detection engine — combining Isolation Forest anomaly detection with Random Forest classification — that is purpose-built for real-time zero-day DoS exploit detection, with DoS-domain-specific feature engineering, a DoS-focused real-time simulation pipeline, and evaluation against DoS-specific performance metrics and baselines.

PHASE 2: DATASET ACQUISITION AND DOMAIN SCOPING
2.1 Dataset: CICIDS2017
The Canadian Institute for Cybersecurity Intrusion Detection System 2017 (CICIDS2017) dataset is the primary dataset for this project. It was generated over five days in a live network environment and contains realistic labeled network flow records for both benign traffic and multiple attack categories, including several DoS and DDoS attack types.
Download Source:
https://www.unb.ca/cic/datasets/ids-2017.html
Specific Files to Download (CSV format — MachineLearningCSV folder):
    • Friday-WorkingHours-Afternoon-DDos.pcap_ISCX.csv — DDoS attack traffic
    • Friday-WorkingHours-Afternoon-PortScan.pcap_ISCX.csv
    • Friday-WorkingHours-Morning.pcap_ISCX.csv — benign traffic
    • Thursday-WorkingHours-Afternoon-Infilteration.pcap_ISCX.csv
    • Wednesday-workingHours.pcap_ISCX.csv — contains DoS Hulk, DoS GoldenEye, DoS Slowloris, DoS Slowhttptest, Heartbleed
Most Important File for This Project:
Wednesday-workingHours.pcap_ISCX.csv — this single file contains DoS Hulk, DoS GoldenEye, DoS Slowloris, DoS Slowhttptest, and Heartbleed attack flows alongside benign traffic. This is your primary working dataset.
