# ARCHITECTURE_DECISIONS.md

# ADR-001

Decision:

Use Python as the primary implementation language.

Reason:

Strong ML ecosystem and cybersecurity tooling support.

Status:

Approved

---

# ADR-002

Decision:

Use CICIDS2017 as primary dataset.

Reason:

Contains realistic DoS attack scenarios and benign traffic.

Status:

Approved

---

# ADR-003

Decision:

Use Isolation Forest for anomaly detection.

Reason:

Efficient unsupervised anomaly detection suitable for zero-day identification.

Status:

Approved

---

# ADR-004

Decision:

Use Random Forest for attack classification.

Reason:

High accuracy and interpretability.

Status:

Approved

---

# ADR-005

Decision:

Use flow-based features.

Reason:

Scalable and computationally efficient.

Status:

Approved

---

# ADR-006

Decision:

Use simulated real-time streaming.

Reason:

Academic constraints prevent collection of live zero-day traffic.

Status:

Approved

---

# ADR-007

Decision:

Use Streamlit dashboard.

Reason:

Rapid implementation and strong visualization capability.

Status:

Approved
